# **QA Portfolio — Test Cases, Checklists, Bug Reports & Automation Concepts**

This repository is a **single, organized hub** for all of my QA work across personal, academic, and open‑source projects.  
It includes **manual testing artifacts**, **structured checklists**, **bug reports**, and **automation proof‑of‑concepts** written entirely in Markdown for clarity and reproducibility.

Even when the code samples aren’t executable, they demonstrate **testing logic**, **workflow design**, and **how I approach QA as a discipline**.

---

## **What This Repo Contains**

### **1. Test Cases & Test Suites**  
Structured manual test cases covering:  
- Functional testing  
- UI/UX validation  
- Regression scenarios  
- Edge‑case exploration  
- Risk‑based prioritization  

Each test case follows a consistent, readable format.

---

### **2. QA Checklists**  
Reusable, scenario‑driven checklists for:  
- Web apps  
- APIs  
- Installers & onboarding flows  
- Blender / creative tools  
- Accessibility  
- Performance sanity checks  
- Release‑readiness  

These checklists reflect real QA habits: modular, repeatable, and adaptable.

---

### **3. Bug Reports**  
A curated set of bug reports from:  
- My own projects  
- Open‑source contributions  
- Tools and apps I’ve tested  
- Reproductions of issues I’ve helped identify  

Each report includes:  
- Steps to reproduce  
- Expected vs. actual behavior  
- Environment details  
- Severity & impact  
- Attachments (screenshots, logs, repro files when applicable)

---

### **4. Automation Proof‑of‑Concepts**  
Markdown‑based examples of:  
- Pseudocode for automated tests  
- Example test flows  
- CI‑friendly folder structures  
- Pre‑commit hook logic  
- How I would structure automation for future projects  

These are **not executable**, but they show my thinking, structure, and approach to scalable QA.

---

## 🧭 **Why This Repo Exists**

I’m building a career in QA with a focus on:  
- **Reproducible workflows**  
- **Open‑source contribution etiquette**  
- **Scenario‑driven testing**  
- **Clear communication**  
- **Modular documentation**  

This repo acts as a **public portfolio** of how I think, test, and structure quality work.
